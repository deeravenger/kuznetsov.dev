Liltag - Простое Решение Для Управления Тегами На Вашем Сайте

Я продолжаю развивать свой open-source проект Econumo и однажды мне понадобилось добавлять аналитические скрипты на публичную версию сайта, при этом не женить их в репозитории.

Можно было бы использовать отдельный шаблон для продакшена, но проблема в том, что я предоставляю единый образ с уже собранной и минифицированной статикой. Это делает замену шаблона неудобной.

Альтернативный вариант - использовать Google Tag Manager или аналогичные сервисы. Однако такие системы довольно сложны и могут не соответствовать требованиям GDPR, что не подходит для некоторых пользователей open-source версии.

В итоге, я решил создать собственную упрощенную версию Tag Manager, которую можно гибко настраивать через JSON-конфигурацию. Это решение вам пригодится как лично мне, так и для open-source сообщества.

Так родился Liltag.

Всего за 20 минут

Решение о создании такого инструмента пришло мне по дороге из офиса к поезду. Путь домой на поезде занимал примерно 20 минут, и у меня был только телефон, что делало программирование затруднительным. Тем не менее, я решил обратиться за помощью к ChatGPT. После нескольких запросов я получил код, содержимое репозитория и документацию. Настоящая магия!

Первую версию библиотеки я разработал за одну поездку и она заработала уже к моему возвращению домой. В течение следующих нескольких дней мы с ChatGPT улучшали и дорабатывали библиотеку: улучшили программный интерфейс, провели небольшой рефакторинг, добавили автоматическую сборку с помощью Github Actions и обновили документацию.

Примерно после трех поездок на поезде и окончательного тестирования, я опубликовал библиотеку в open-source.

Самым сложным было определить функциональные возможности, так как хотелось сделать простое, но в то же время гибкое решение. Мне кажется, что это удалось. Что касается качества кода, это поле для дальнейших улучшений, но и сейчас он уже на высоком уровне.

Возможности Liltag

  • Загрузка скриптов, стилей и HTML-компонентов на ваш сайт по аналогичному принципу, что и Google Tag Manager.
  • В конфигурации можно указать, в какую часть страницы будет вставлен тег (head, body top, или body bottom).
  • Тег инициализируется в ответ на выбранное событие (подробнее об этом ниже).

Установка Liltag

Простое подключение Liltag

<script src="//deeravenger.github.io/liltag/dist/liltag.min.js" defer></script>
<script>
  const lilTag = new LilTag("path_or_url/to/liltag_config.json");
  lilTag.init();
</script>

Вы можете скачать liltag.min.js и подключить его локально.

Асинхронное подключение Liltag

<script>
  (function () {
    const script = document.createElement("script");
    script.src = "//deeravenger.github.io/liltag/dist/liltag.min.js";
    script.async = true;
    script.onload = function () {
      const lilTag = new LilTag("path_or_url/to/liltag_config.json");
      lilTag.init();
    };
    document.head.appendChild(script);
  })();
</script>

Конфигурация Liltag

Загрузка конфига по URL

При установке Liltag, вы можете указать URL конфига, который будет загружен и использован для инициализации тегов.

<script src="//deeravenger.github.io/liltag/dist/liltag.min.js" defer></script>
<script>
  const lilTag = new LilTag("path_or_url/to/liltag_config.json");
  lilTag.init();
</script>

Пример конфигурации:

{
  "tags": [
    {
      "id": "analytics",
      "trigger": "pageLoad",
      "content": "<script type=\"text/javascript\">console.log('Analytics script loaded.');</script>",
      "location": "head"
    }
  ]
}

Конфигурация Liltag в коде

const lilTag = new LilTag({
  "tags": [
    {
      "id": "analytics",
      "trigger": "pageLoad",
      "content": "<script type=\"text/javascript\">console.log('Analytics script loaded.');</script>",
      "location": "head"
    }
  ]
});

lilTag.init();

Кэширование конфига

Liltag может кэшировать конфигурацию, что позволяет избежать лишних запросов к серверу и улучшает производительность.

const lilTag = new LilTag('path_or_url/to/liltag_config.json');
lilTag.enableCache(7200); // Включить кэширование с TTL 2 часа (7200 секунд)
lilTag.init();

Полный конфиг

Каждый тег в конфиге может содержать следующие поля:

  • id: Уникальный идентификатор тега.
  • trigger: Событие, которое инициирует тег. Доступны следующие события:
    • pageLoad: Инициализация при загрузке страницы.
    • domReady: Инициализация при готовности DOM.
    • timeDelay: Инициализация с задержкой в секундах.
    • elementVisible: Инициализация при видимости элемента на странице.
    • customEvent: Инициализация на произвольное событие.
  • content: HTML-контент, который будет вставлен на страницу.
  • location: Позиция для вставки тега:
    • head: В head.
    • bodyTop: В верхнюю часть body.
    • bodyBottom: В нижнюю часть body.
  • delay (optional): Задержка инициализации в секундах. Используется, если триггером является timeDelay.
  • selector (optional): Селектор элемента, по которому инициируется тег. Используется, если триггером является elementVisible.
  • eventName (optional): Имя события, по которому инициируется тег. Используется, если триггером является customEvent.
